# 03. 5G System Architecture and Core Functions

To deliver high speeds, ultra-low latencies, and massive connections, the 5G System is physically and logically partitioned into highly specialized building blocks. This guide explores the 3GPP architectural blueprints, key network functions, and interfaces that make end-to-end 5G communication a reality.

![[Materials/Slides/System_Slide_1.png]]

---

## 🏗️ The 3 Main Blocks of the 5G System

If you pick up a 5G phone and search Google, you are initiating an end-to-end signal and data flow across the network. The high-level architecture is divided into three major building blocks:

$$\text{User Equipment (UE)} \longrightarrow \mathbf{\text{Access Network (5G RAN)}} \longrightarrow \mathbf{\text{5G Core Network (5GC)}} \longrightarrow \text{Data Network (Internet)}$$

![[Materials/Slides/System_Slide_2.png]]

### 1. User Equipment (UE)
This is the user's terminal device, such as a smartphone, connected laptop, smart utility meter, logistics tracker, or autonomous vehicle.

### 2. 5G Radio Access Network (5G RAN)
The RAN provides wireless, over-the-air connectivity between the UE and the network.
* **Technology:** Built on **New Radio (NR)** access technology.
* **Base Stations:** 
  * **gNodeB (Next-Generation NodeB):** The standard 5G base station providing NR user plane and control plane connectivity.
  * **ng-eNB (Next-Generation eNodeB):** An upgraded 4G LTE base station that connects to the 5G Core Network, supporting dual-connectivity configurations.

### 3. 5G Core Network (5GC)
The central brain of the network. It handles critical control signaling, security, subscription checks, device tracking, and data routing to external networks.
* **Data Network (DN):** The final destination (e.g., the public internet, corporate intranets, or carrier services).

## 🗺️ Architectural Representations (Point-to-Point vs. SBA)

In 3GPP standards, the 5G System architecture is represented in two primary ways:

![[Materials/Slides/System_Slide_4.png]]

### 1. Reference Point Representation (Point-to-Point)
This view shows the direct interfaces connecting individual network functions. It is highly detailed for physical signaling configurations and traditional deployment topologies.
* **Key Interfaces:**
  * **N1:** Reference point for control plane signaling between the UE and the AMF.
  * **N2:** Interface between the gNodeB and the AMF (carrying RAN control signaling).
  * **N3:** Interface between the gNodeB and the UPF (carrying user data plane streams).
  * **N4:** Interface between the SMF and the UPF (carrying session management commands from control to user plane).
  * **N11:** Interface between the AMF and the SMF (coordinating session registration and handovers).
  * **N15:** Interface between the AMF and the PCF (applying geographic or mobility policies to the device registration).

### 2. Service-Based Representation (Service-Based Architecture - SBA)
In this modern approach, core Network Functions (NFs) connect to a common bus/service highway. Instead of point-to-point connections, NFs act as **Service Producers** or **Service Consumers** communicating via standard HTTP/2 REST APIs.
* **SBA Interfaces:** Represented by the letter **N** followed by the function name in lowercase:
  * **Namf:** Service interface exposed by the AMF.
  * **Nsmf:** Service interface exposed by the SMF.
  * **Nudm:** Service interface exposed by the UDM.
  * **Npcf:** Service interface exposed by the PCF.

## ⚡ The Ultimate Golden Rule: Control & User Plane Separation (CUPS)

Introduced as a foundational principle in 3GPP releases, CUPS strictly decouples the path of signaling from the path of user data.

![[Materials/Slides/System_Slide_5.png]]

* **The User Plane (Red Path):** Transports actual user data payloads (e.g., streaming video packets, loaded web pages, or haptic control signals).
  * *Flow:* **UE ➔ gNodeB ➔ UPF ➔ Data Network (Internet)**
* **The Control Plane (Green Path):** Transports network signaling traffic (e.g., device registration, authentication, session establishment, security keys, or cell tower handovers).
  * *Flow:* **UE ➔ gNodeB ➔ AMF ➔ SMF / PCF / UDM / AUSF**

### Why is CUPS Critical?
1. **Independent Scaling for Diverse Use Cases:**
   * **Massive MTC (mMTC):** Millions of smart meters create high control plane signaling overhead when connecting, but transmit very little user plane data. Operators can scale just the **Control Plane** (AMF/SMF) without paying for idle high-capacity data pipes.
   * **Mobile Broadband (eMBB):** Smartphones stream massive amounts of 4K data, requiring a massive User Plane path, even though device numbers are lower. Operators can scale just the **User Plane** (UPF) data routing capacity.
2. **Geographical and Performance Flexibility:** Operators can pool complex Control Plane functions in a single centralized, easy-to-manage data center, while pushing User Plane functions (UPFs) to the **edge of the network** (closer to the user) to drastically reduce latency (critical for URLLC).

## 🎛️ Inside the Box: Key 5G Core Network Functions

![[Materials/Slides/System_Slide_6.png]]

The 5G Core is divided into dedicated microservices:

### 📡 5G RAN (gNodeB / Ng-eNB)
Provides the wireless physical layer and radio resource access interface for User Equipment.
* *Primary Responsibility:* **Wireless Connectivity**

![[Materials/Slides/System_Slide_7.png]]

### 🚀 UPF (User Plane Function)
The high-speed router of the user plane, passing all data payloads to the internet.
* *Primary Responsibilities:* **Mobility Anchor** (ensuring your connection doesn't drop when switching cells), **Policy Enforcement** (applying caps or throttling), and **Lawful Interception**.

![[Materials/Slides/System_Slide_8.png]]

### 🔀 SMF (Session Management Function)
Manages the lifecycle of user data sessions.
* *Primary Responsibilities:* **Session Management**, IP address allocation, PDU session establishment, modification, and release.

![[Materials/Slides/System_Slide_9.png]]

### 📲 AMF (Access and Mobility Management Function)
The gateway for all control plane signaling.
* *Primary Responsibilities:* **Mobility Management**, **Registration**, connection state tracking, ciphering, and integrity protection of NAS (Non-Access Stratum) signaling.

![[Materials/Slides/System_Slide_10.png]]

### 🔒 AUSF (Authentication Server Function)
Acts as a security vault verifying the device's identity.
* *Primary Responsibility:* **Security**, performing authentication algorithms (e.g., 5G-AKA or EAP-AKA') to authenticate the UE during initial registration.

![[Materials/Slides/System_Slide_11.png]]

### 💾 UDM & UDR (Unified Data Management / Repository)
The primary user registry database. The UDR houses raw user subscription details, while the UDM serves as the front-end manager.
* *Primary Responsibility:* **User Subscription Data** (checking roaming rights, active data profiles, and SIM card blocks).

![[Materials/Slides/System_Slide_12.png]]

### ⚖️ PCF (Policy Control Function)
The rule enforcer of the system.
* *Primary Responsibilities:* **Session Management Policies**, **Non-session Management Policies**, and **User Charging Rules** (interfaces with billing systems).
