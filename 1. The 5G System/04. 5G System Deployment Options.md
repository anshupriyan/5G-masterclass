# 04. 5G System Deployment Options

Deploying 5G globally is a massive financial and technical challenge. Mobile operators have invested billions of dollars in their existing 4G LTE infrastructure. Tearing it down overnight is impossible. To bridge this migration gap, the 3GPP standardization body defined a set of **Deployment Options (Options 1 through 7)** that allow 4G and 5G to coexist, interwork, and transition smoothly.

![[Materials/Slides/Deployment_Slide_1.png]]

---

## 🏗️ Standalone (SA) vs. Non-Standalone (NSA)

Before examining the specific options, we must define the two primary deployment pathways:

![[Materials/Slides/Deployment_Slide_10.png]]

* **Standalone (SA):** The network uses only **one generation of technology** across both the radio access network and the core network. It is entirely independent. (e.g., Option 1 for pure 4G, Option 2 for pure 5G).
* **Non-Standalone (NSA):** The network uses a **combination of both 4G and 5G technologies** simultaneously. The existing 4G LTE network serves as a signaling anchor to deploy 5G coverage quickly without requiring an immediate core upgrade.

### Multi-Radio Dual Connectivity (MR-DC)
In NSA configurations, the User Equipment (UE) connects to two different base stations at the same time:
* **Master Node (MN):** The base station that handles the primary control plane signaling connection to the Core Network.
* **Secondary Node (SN):** The secondary base station acting as an additional data pipe to boost user plane speeds.

## 🔍 The 3GPP Deployment Options

---

### 1. The Pure Architectures: Option 1 and Option 2 (Standalone)

![[Materials/Slides/Deployment_Slide_2.png]]

* **Option 1 (Legacy 4G Standalone):**
  * *Setup:* Connects a 4G base station (**eNodeB**) directly to a 4G Core network (**EPC** — Evolved Packet Core).
  * *Status:* The legacy baseline.
* **Option 2 (Pure 5G Standalone):**
  * *Setup:* Connects a standalone 5G base station (**gNodeB**) directly to the native **5G Core (5GC)**.
  * *Status:* The ultimate target architecture.
  * *Why it matters:* There is zero reliance on legacy 4G. This is the **only option** that allows operators to fully unlock advanced 5G capabilities such as Ultra-Reliable Low-Latency Communications (URLLC), Massive Machine-Type Communications (mMTC), and Network Slicing.

---

### 2. The Early Adopter Path: Option 3 (Non-Standalone)

When 5G was first launched globally, almost every major operator used **Option 3** (specifically known as **EN-DC**: E-UTRAN New Radio Dual Connectivity).

![[Materials/Slides/Deployment_Slide_3.png]]

* **The Setup:** The network uses a legacy **4G Core (EPC)**. The smartphone connects to a 4G eNodeB (Master Node) and a 5G gNodeB (Secondary Node).
* **How it works:** All control signaling (Control Plane) goes through the 4G Master Node to the 4G Core. The 5G Secondary Node is used purely as a high-capacity data booster (User Plane).
* **Why operators love it:** It allows them to market "5G speeds" incredibly fast and cheaply without upgrading their core network infrastructure right away. 
* **The Downside:** It cannot support true ultra-low latency or massive IoT use cases because the core brain is still legacy 4G.

#### 🔧 Option 3 Sub-Options (3, 3a, 3x)
To manage user plane data flows, 3GPP defined three distinct sub-options:

![[Materials/Slides/Deployment_Slide_4.png]]

* **Option 3:**
  * *Split Point:* **4G eNodeB (Master Node)**.
  * *Data Flow:* All user data from the 4G Core (EPC) goes to the 4G eNodeB, which splits the traffic and routes a portion over a direct interface to the 5G gNodeB.
* **Option 3a:**
  * *Split Point:* **4G Core (EPC)**.
  * *Data Flow:* The 4G Core has direct user plane connections to *both* the 4G eNodeB and 5G gNodeB. Data is split at the core level; there is no direct data interface between the base stations.
* **Option 3x (The Industry Standard):**
  * *Split Point:* **5G gNodeB (Secondary Node)**.
  * *Data Flow:* User data from the 4G Core goes directly to the 5G gNodeB. The 5G base station handles the split, routing a portion to the UE and the rest over to the 4G eNodeB. 
  * *Why 3x is preferred:* 5G base stations have much faster, modern processors to handle the complex traffic splitting algorithms compared to older 4G base stations.

---

### 3. Option 4 - NE-DC (5G-Dominant Non-Standalone)

![[Materials/Slides/Deployment_Slide_5.png]]

* **The Setup:** Uses the new **5G Core (5GC)**. The 5G gNodeB acts as the Master Node (handling control signaling), while a legacy 4G eNodeB acts as the Secondary Node to supplement data speeds.
* **Why it matters:** A migration option when 5G radio coverage is mature and operators want to use their 5G Core to run signaling while still leveraging legacy 4G cells for extra data pipes.

#### 🔧 Option 4 Sub-Options (4 and 4a)

![[Materials/Slides/Deployment_Slide_6.png]]

* **Option 4:** User plane data from the 5G Core is routed to the 5G gNodeB, which splits the traffic and routes a portion over to the 4G eNodeB.
* **Option 4a:** The 5G Core has direct user plane connections to *both* the 5G gNodeB and the 4G eNodeB, splitting the traffic at the core level.

---

### 4. Option 5 - eLTE Standalone

![[Materials/Slides/Deployment_Slide_7.png]]

* **The Setup:** Connects an upgraded 4G base station (called enhanced LTE or **eLTE eNodeB**) directly to the new **5G Core (5GC)**.
* **Why it matters:** Allows an operator to upgrade their core network brain to 5G first, running the new system functions over their existing 4G radio network without deploying new 5G base stations immediately.

---

### 5. Option 6 - Standalone 5G Radio to 4G Core (NOT SUPPORTED)

![[Materials/Slides/Deployment_Slide_8.png]]

* **The Proposed Setup:** Connects a standalone 5G base station (gNodeB) directly back to a legacy 4G Core (EPC) without any 4G radio interworking.
* **Status:** **Not supported by 3GPP**. It was determined that building 5G terminal devices to support this configuration was technically inefficient and offered no commercial benefit, so it was dropped from the standards.

---

### 6. Option 7 - NG-EN-DC (Core-First Migration)

![[Materials/Slides/Deployment_Slide_9.png]]

* **The Setup:** Uses the **5G Core (5GC)**. The Master Node for control plane signaling is an upgraded 4G base station (eLTE eNodeB), while the Secondary Node for user plane data is the 5G gNodeB.
* **Why it matters:** Designed for operators who have upgraded to a 5G Core but still have patchy 5G radio coverage, relying on their mature 4G footprint to handle signaling anchors.

---

## 📊 Summary Comparison

| Deployment Option | Architecture Type | Master Node (Signaling) | Secondary Node (Data) | Core Network | Main Purpose |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Option 1** | Standalone (4G) | eNodeB | *None* | 4G EPC | Legacy baseline |
| **Option 2** | **Standalone (5G)** | **gNodeB** | **None** | **5G Core** | **Full 5G capabilities (URLLC/mMTC)** |
| **Option 3 (EN-DC)** | Non-Standalone | eNodeB | gNodeB | 4G EPC | Fast, low-cost marketing deployment (splits: 3/3a/3x) |
| **Option 4** | Non-Standalone | gNodeB | eNodeB | 5G Core | 5G-centric migration (splits: 4/4a) |
| **Option 5** | Standalone (eLTE) | eLTE eNodeB | *None* | 5G Core | Core upgrade first, using 4G radio |
| **Option 6** | *Not Supported* | gNodeB | *None* | 4G EPC | *Dropped from standards* |
| **Option 7 (NG-EN-DC)** | Non-Standalone | eLTE eNodeB | gNodeB | 5G Core | Core-first migration with patchier 5G coverage |
