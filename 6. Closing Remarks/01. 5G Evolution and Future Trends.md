# 01. 5G Evolution and Future Trends

As 5G deployments mature, the wireless industry is actively executing a dual-track roadmap: commercializing **5G-Advanced** (governed by 3GPP Releases 18 and 19) while defining the foundational architectures, spectrum requirements, and disruptive technologies for **6G (Sixth Generation)** mobile networks targeting deployment around 2030.

---

## 🧠 1. AI/ML Integration in 5G Networks

Artificial Intelligence (AI) and Machine Learning (ML) are evolving from auxiliary management tools into core architectural components of both the **Radio Access Network (RAN)** and the **5G Core (5GC)**.

### A. Intelligent RAN (AI in the Air Interface)
In traditional RANs, schedulers and beamformers rely on deterministic, heuristic algorithms. 5G-Advanced introduces native AI/ML-driven optimization blocks at the physical and MAC layers:
* **Predictive Handovers:** Rather than waiting for a UE to drop below a strict RSRP threshold (A3 event) to trigger a handover, deep learning models (such as LSTM networks or Transformer-based trajectory models) predict device mobility paths. The network preemptively prepares the target gNodeB and reserves resources, cutting handover interruption time close to 0 ms.
* **Dynamic Beam Tracking & Management:** Massive MIMO requires continuous beam adjustment. Reinforcement Learning (RL) agents dynamically track UE movement in 3D space, predicting angular velocity to focus beam energy precisely, reducing control plane beam-sweeping overhead and mitigating blockage.
* **Intelligent Scheduling & Link Adaptation:** Deep learning models predict fast fading channel states based on historical Channel State Information (CSI) feedback. This allows the MAC scheduler to select the optimal Modulation and Coding Scheme (MCS) more accurately than standard lookup tables, maximizing spectral efficiency.
* **RAN Energy Saving:** AI models analyze traffic load patterns in real time, predicting low-demand windows to dynamically put transceiver channels or entire cells into deep sleep modes (MIMO sleep, carrier shutdown) without degrading local QoS.

### B. Intelligent Core: The NWDAF (Network Data Analytics Function)
The **NWDAF (Network Data Analytics Function)**, defined in 3GPP TS 29.520, is the standardized analytics engine of the 5G Core Service-Based Architecture. It interfaces with all control plane NFs via HTTP/2 REST services to gather data, train models, and provide predictive intelligence.

```
                  ┌───────── Gather Data (HTTP PUT/POST) ────────┐
                  ▼                                              │
  ┌──────────────────────────────┐                       ┌──────────────┐
  │  Core Network Functions      │                       │    NWDAF     │
  │ (AMF, SMF, UDM, PCF, NRF...) │                       │  (Analytics) │
  └──────────────────────────────┘                       └──────────────┘
                  ▲                                              │
                  └───────── Request Predictions (HTTP GET) ─────┘
```

#### NWDAF Services and Interfaces:
* **`Nnwdaf_EventsSubscription`:** Allows NFs (like the PCF or AMF) to subscribe to specific analytics reports (periodic or threshold-triggered).
* **`Nnwdaf_AnalyticsInfo`:** Allows NFs to query real-time or historical predictive analytics directly.

#### Key Use Cases of NWDAF:
1. **Network Slice Load Analytics:** Predicts the load level of specific network slices (S-NSSAI). If NWDAF predicts a slice will experience congestion in 10 minutes, the NSSF/AMF can dynamically redirect new PDU session requests to alternative slices or trigger resource scaling.
2. **UE Mobility Analytics:** Analyzes historical Tracking Area Updates (TAUs) and cell camping transitions to predict a UE's future location profile. The AMF uses this to optimize paging strategies (e.g., paging only in predicted cells rather than the entire Tracking Area).
3. **Network Performance Analytics:** Predicts user-plane QoS metrics (throughput, packet delay, packet loss). The PCF uses these predictions to adjust PCC rules dynamically, adapting QoS parameters before user experience degrades.
4. **Abnormal Behavior Detection:** Detects security threats or malfunctioning devices (e.g., DDoS attacks, signaling storms, IMSI catchers) by identifying anomalies in NAS signaling and UPF traffic flows.

## 🚀 2. 5G-Advanced (3GPP Releases 18 & 19)

**5G-Advanced** marks the second phase of 5G standardization, enhancing the system's capabilities to support demanding industrial, commercial, and interactive use cases.

### A. Extended Reality (XR) Optimizations
XR applications (AR, VR, Mixed Reality) impose asymmetric, high-bandwidth, and low-latency demands. 5G-Advanced optimizes the RAN for XR via:
* **Application-Aware Scheduling:** The gNodeB scheduler reads XR traffic characteristics (frame rates, jitter bounds, burstiness) to prioritize XR packets, keeping latency jitter minimal.
* **Power-Saving Modes:** Optimizes Discontinuous Reception (DRX) sleep cycles to align with the arrival of video frames from the application layer, extending device battery life.

### B. Sidelink Enhancements (Direct UE-to-UE Communication)
Building upon early V2X (Vehicle-to-Everything) designs, Releases 18/19 extend Sidelink (PC5 interface) capabilities:
* **Sidelink Relay:** Allows a device with a strong cellular link (e.g., a vehicle or a mounted CPE) to act as a layer-2 or layer-3 relay for out-of-coverage devices (e.g., wearables or remote sensors), bridging them to the gNodeB.
* **Unlicensed Spectrum Sidelink:** Operates Sidelink over shared unlicensed bands (5 GHz and 6 GHz), expanding capacity for local ad-hoc communication.

### C. Non-Terrestrial Networks (NTN) Evolution
5G-Advanced achieves deep satellite-to-cellular integration:
* **Handover & Mobility:** Standardizes seamless handovers between terrestrial cells and Non-Terrestrial Networks (LEO, MEO, GEO satellites).
* **Doppler & Delay Compensation:** Introduces advanced timing advance (TA) and frequency offset pre-compensation tracking algorithms at the UE and satellite node to resolve massive propagation delays (up to tens of milliseconds) and high Doppler shifts.

### D. RedCap (Reduced Capability) / NR-Light Phase 2
Releases 18/19 evolve RedCap to bridge the gap between high-speed eMBB and low-speed LPWAN (like NB-IoT):
* **Bandwidth Reduction:** Constrains device bandwidth further to **20 MHz** in FR1 and **100 MHz** in FR2.
* **Single Receive Antenna:** Minimizes RF front-end complexity and cost, making it ideal for smartwatches, surveillance cameras, and industrial IoT sensors while maintaining 5G security, low latency, and network slicing.

## 🛰️ 3. Towards 6G: Vision and Technologies

Targeted for initial commercial deployment by **2030**, 6G represents a paradigm shift where the network acts not just as a pipeline for data, but as a unified sensor and intelligent platform.

### A. 6G Key Performance Indicators (KPIs)
Based on early ITU-R IMT-2030 draft visions, 6G will scale performance metrics by an order of magnitude:

| Performance Metric | 5G (IMT-2020) Target | 6G (IMT-2030) Expected Target |
| :--- | :--- | :--- |
| **Peak Data Rate** | 20 Gbps (Downlink) | **1 Tbps** |
| **User Experienced Data Rate** | 100 Mbps | **1 Gbps** |
| **User Plane Latency** | 1 ms (URLLC) | **< 0.1 ms (100 microseconds)** |
| **Connection Density** | $10^6$ devices / $\text{km}^2$ | **$10^7$ devices / $\text{km}^2$** |
| **Reliability** | $99.999\%$ | **$99.99999\%$** |
| **Spectrum Efficiency** | $1\times$ | **$3\times$ to $5\times$** |
| **Positioning Accuracy** | Meter-level (10 cm indoor) | **Centimeter-level (sub-1 cm)** |
| **Mobility Support** | 500 km/h | **1000 km/h** (Supports Hyperloop/Aircraft) |

### B. Disruptive 6G Enabling Technologies

```
                  ┌──────────────────────────────────────────────┐
                  │            6G Disruptive Technologies        │
                  └──────────────────────┬───────────────────────┘
         ┌───────────────────────────────┼───────────────────────────────┐
         ▼                               ▼                               ▼
 ┌───────────────┐               ┌───────────────┐               ┌───────────────┐
 │ Terahertz Spectrum │          │     RIS       │               │     ISAC      │
 │ (100GHz - 3THz)   │          │ (Meta-Surfaces)│               │ (Radar+Comm)  │
 └───────────────┘               └───────────────┘               └───────────────┘
```

#### 1. Terahertz (THz) Communication (100 GHz to 3 THz)
* **Objective:** Unlocks vast, unallocated spectrum blocks to deliver multi-gigabit and terabit-per-second backhaul and short-range access links.
* **Challenges:** Suffers from extreme atmospheric absorption, molecular resonance (especially from water vapor), and very high free-space path loss. This necessitates the use of massively dense, ultra-narrow beam antenna arrays and dynamic line-of-sight tracking.

#### 2. Reconfigurable Intelligent Surfaces (RIS)
* **Objective:** Rather than treating the physical environment (walls, buildings) as passive obstacles, RIS turns them into active components of the network.
* **Mechanism:** RIS consists of arrays of thousands of tiny, low-cost passive electromagnetic scattering elements (meta-surfaces). An external controller dynamically programs the phase shifts of these elements to reflect, refract, or focus incoming radio waves, steering signals around corners to bypass physical blockages without requiring power-hungry active amplifiers.

#### 3. Integrated Sensing and Communication (ISAC)
* **Objective:** Merges wireless communication and radar imaging/sensing into a single hardware platform and unified waveform.
* **Mechanism:** The 6G base station transmits a dual-purpose radio signal. As the signal reflects off objects, the receiver decodes the backscatter to map the physical environment (detecting positions, velocities, shapes, and materials of cars, people, and obstacles) while simultaneously transmitting data. This enables centimeter-level localization and radar-mapping without dedicated radar hardware.

#### 4. Semantic and Goal-Oriented Communication
* **Objective:** A fundamental departure from Shannon's classical information theory, which focuses on transmitting raw bits accurately regardless of meaning.
* **Mechanism:** Semantic communication uses shared AI knowledge bases at the transmitter and receiver. The transmitter extracts the core "meaning" or "intent" of the data (e.g., semantic features of an image or text) and transmits only this compressed representation. The receiver uses its local AI model to reconstruct the full context, reducing the required transmission bandwidth by up to $90\%$ without losing utility.

#### 5. Space-Air-Ground-Ocean Integrated Network (SAGIN)
* **Objective:** Unifies multi-dimensional networks to achieve true global coverage, reaching remote oceans, high altitudes, deserts, and polar zones.
* **Mechanism:** Integrates Low-Earth Orbit (LEO) satellite constellations, High-Altitude Platform Stations (HAPS, such as solar drones or airships), Unmanned Aerial Vehicles (UAVs), terrestrial gNodeBs, and maritime communication links into a single, dynamically routed mesh network.

## 📖 4. Curated 3GPP Reference Standards

To master the 5G system architecture and track its evolution, the following **3GPP Technical Specifications (TS)** are the definitive sources of reference:

| Specification Number | Domain | Description |
| :--- | :--- | :--- |
| **[[TS 23.501]]** | Core / System | **System Architecture for the 5G System (5GS)**<br>Defines the Service-Based Architecture (SBA), NFs (AMF, SMF, UPF), interfaces, QoS model, slicing, and session continuity rules. |
| **[[TS 23.502]]** | Core / Signaling | **Procedures for the 5G System (5GS)**<br>Specifies the signaling sequences for Registration, PDU Session Establishment, Deregistration, and Handovers. |
| **[[TS 38.300]]** | RAN | **NR; NR and NG-RAN Overall Description; Stage 2**<br>Provides the complete functional overview of the Radio Access Network, protocol stack, and multi-connectivity. |
| **[[TS 38.401]]** | RAN / Architecture | **NG-RAN; Architecture Description**<br>Defines the internal split architecture of the gNodeB (CU-DU split, CP-UP split over F1-C and F1-U interfaces). |
| **[[TS 38.211]]** | RAN / PHY | **NR; Physical Channels and Modulation**<br>Specifies physical channels (PDSCH, PUSCH, PDCCH, PUCCH), frame structure, flexible numerologies, resource grids, and MIMO precoding. |
| **[[TS 38.331]]** | RAN / RRC | **NR; Radio Resource Control (RRC) Protocol Specification**<br>Defines RRC state machines (Idle, Connected, Inactive), system information blocks (MIB, SIB1), and RRC signaling messages. |
| **[[TS 38.321]]** | RAN / MAC | **NR; Medium Access Control (MAC) Protocol Specification**<br>Covers logical-to-transport mapping, dynamic scheduling, HARQ processes, Random Access (Msg1-Msg4), and MAC Control Elements. |
| **[[TS 38.322]]** | RAN / RLC | **NR; Radio Link Control (RLC) Protocol Specification**<br>Details the TM, UM, and AM transfer modes, segmentation/reassembly, and ARQ window operation. |
| **[[TS 38.323]]** | RAN / PDCP | **NR; Packet Data Convergence Protocol (PDCP) Specification**<br>Covers header compression (RoHC), security ciphering, integrity protection, split bearer routing, and reordering. |
| **[[TS 37.324]]** | RAN / SDAP | **NR; Service Data Adaptation Protocol (SDAP) Specification**<br>Specifies QoS flow-to-data radio bearer mapping, explicit and reflective QoS marking, and SDAP header formats. |
| **[[TS 29.500]]** | Core / SBA | **Technical Realization of Service Based Architecture; Stage 3**<br>Details HTTP/2, JSON payloads, serialization, and TCP connection rules for service-based APIs. |
| **[[TS 29.244]]** | Core / User Plane | **Interface between Control Plane and User Plane Nodes (PFCP)**<br>Specifies the Packet Forwarding Control Protocol (PFCP) over the N4 interface, detailing PDR, FAR, BAR, QER, and URR definitions. |
