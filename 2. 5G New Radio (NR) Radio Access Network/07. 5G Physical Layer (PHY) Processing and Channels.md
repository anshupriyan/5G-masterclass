# 07. 5G Physical Layer (PHY) Processing and Channels

The Physical Layer is **Layer 1** of the 5G NR protocol stack. It receives processed data units called **Transport Blocks** from the MAC layer and maps them onto a set of physical time-frequency resources allocated for transmission over the airwaves.

![[PHY_Slide_1.png]]

---

## 🩻 1. The Core Physical Channels Inventory

5G NR PHY handles communication via specific physical channels across both the downlink (network to phone) and uplink (phone to network):

![[PHY_Slide_2.png]]

![[PHY_Slide_3.png]]

### Downlink Channels:
* **PBCH (Physical Broadcast Channel):** Carries the essential system parameters (Master Information Block [MIB]) needed for a device to initially sync, timing-align, and access the cell. Maps from the **BCH** transport channel.
* **PDSCH (Physical Downlink Shared Channel):** The primary user data channel. It carries the downlink application payload, user traffic, and paging messages. Maps from the **DL-SCH** and **PCH** transport channels.
* **PDCCH (Physical Downlink Control Channel):** Carries **DCI (Downlink Control Information)** specifying downlink scheduling assignments, uplink resource grants, HARQ parameters, and power-control instructions.

### Uplink Channels:
* **PUSCH (Physical Uplink Shared Channel):** The main uplink data channel carrying user payloads, application traffic, and optionally uplink control feedback. Maps from the **UL-SCH** transport channel.
* **PUCCH (Physical Uplink Control Channel):** Carries **UCI (Uplink Control Information)**, including Hybrid-ARQ acknowledgements (ACK/NACK), scheduling requests (SR), and Channel State Information (CSI) reports.
* **PRACH (Physical Random Access Channel):** Carries random-access preambles used by a device to transition from an idle state to an active connection, enabling initial sync and timing advance calculations. Maps from the **RACH** transport channel.

## ⚙️ 2. Step-by-Step Processing Timeline of a Transport Block

When a **Transport Block (TB)** arrives from the MAC layer to be sent over the **Downlink Shared Channel (DL-SCH)**, it goes through a sequence of mathematical processing stages within the PHY layer:

![[PHY_Slide_4.png]]

---

### Step 1: Transport Block CRC Attachment
To ensure error detection at the destination, a **CRC (Cyclic Redundancy Check)** sequence is calculated and attached directly to the end of the incoming Transport Block.

![[PHY_Slide_5.png]]

* **Mechanisms:** Provides block-level verification (`TB-CRC`) to guarantee the integrity of the overall packet. If the CRC check fails at the receiver, a NACK is generated in the HARQ loop.

---

### Step 2: Channel Coding Graph Selection & Code Block Segmentation

#### A. LDPC Graph Selection
5G NR uses **LDPC (Low-Density Parity-Check)** codes for data channels, replacing the Turbo codes utilized in 4G LTE. The layer chooses between two distinct base graphs based on the total Transport Block Size (TBS) and code rate ($R$):

![[PHY_Slide_6.png]]

* **Base Graph 1 (BG1):** Optimized for large transport blocks and high data rates ($TBS > 3824$ bits, or $TBS > 292$ bits when code rate $R > 0.67$). BG1 features a matrix size of 46 rows by 68 columns.
* **Base Graph 2 (BG2):** Optimized for smaller transport blocks and lower code rates ($TBS \le 3824$ bits, or $TBS \le 292$ bits).

#### B. Code Block Segmentation
If the Transport Block exceeds the maximum processing size of the selected LDPC graph (e.g., 8448 bits for BG1 or 3840 bits for BG2), it is chopped up into smaller segments called **Code Blocks (CBs)**.

![[PHY_Slide_7.png]]

* **Inner CRC:** An additional inner CRC sequence (`CB-CRC`) is attached to the end of every individual Code Block. This allows the receiver to detect and isolate bit errors within specific code blocks.

---

### Step 3: LDPC Channel Coding
The segmented blocks are fed into the **LDPC Coding** engine.

![[PHY_Slide_8.png]]

* **Parity Bit Addition:** This stage adds structural, redundant parity bits to the data stream. It provides forward error correction capability so the receiver can mathematically repair over-the-air bit flips caused by noise and path loss without requiring immediate retransmissions.

---

### Step 4: Rate Matching
The amount of physical radio resources allocated by the MAC scheduler varies dynamically from slot to slot. The **Rate Matching** engine extracts the exact number of bits needed to match the allocated time-frequency resource grid.

![[PHY_Slide_9.png]]

* **Bit Selection:** It punctures (discards) or repeats coded bits to adapt the code rate precisely to the current transmission slot allocation window, outputting a bitstream that perfectly matches the physical channel's capacity.

---

### Step 5: Scrambling
The output bitstream is mixed with a cell-specific, pseudo-random scrambling code sequence.

![[PHY_Slide_10.png]]

* **Inter-Cell Interference Prevention:** By randomizing the data stream, scrambling prevents persistent inter-cell interference if neighboring towers happen to transmit identical sequences on overlapping frequencies at the same time.

---

### Step 6: Modulation
The scrambled digital bits are grouped and mapped into complex-valued analog data symbols.

![[PHY_Slide_11.png]]

Depending on channel quality, 5G NR dynamically applies different modulation orders in both uplink and downlink:
* **QPSK:** 2 bits per symbol (used in poor radio conditions near cell edges).
* **16QAM:** 4 bits per symbol.
* **64QAM:** 6 bits per symbol (64-state Quadrature Amplitude Modulation).
* **256QAM:** 8 bits per symbol (256-state Quadrature Amplitude Modulation), allowing exceptionally high throughput in clean signal environments near the base station.

---

## 📡 3. Antenna Mapping and Multi-Antenna Precoding

Once modulation symbols are generated, they flow through the multi-antenna spatial processing pipeline:

### Layer Mapping
A single data stream can be split across multiple parallel spatial paths to maximize throughput (spatial multiplexing) or reliability. **Layer Mapping** distributes the modulation symbols across 1 up to a maximum of 8 spatial layers (or codewords) simultaneously.

![[PHY_Slide_12.png]]

### Antenna Port Precoding
The layers are mapped onto **Virtual Antenna Ports** using a mathematical **Precoding Matrix**.

![[PHY_Slide_13.png]]

* **Beamforming:** Precoding applies custom phase and amplitude offsets to the data streams across the antenna array. This enables the base station to focus the radio energy into narrow, highly directed beams focused toward specific users (**Beamforming**), maximizing signal strength and range while reducing overall cell noise.

---

## 🧱 4. Resource and Physical Antenna Mapping

The final stage prepares the precoded data symbols for physical transmission over the air:

### Resource Mapping
The symbols are mapped into their assigned slots within the physical **OFDM (Orthogonal Frequency Division Multiplexing)** resource grid.

![[PHY_Slide_14.png]]

* **Grid Allocation:** Each symbol is assigned to a specific virtual subcarrier (frequency domain) and OFDM symbol index (time domain).

### OFDM, DAC, and RF Modulation
1. **OFDM Signal Generation:** An Inverse Fast Fourier Transform (IFFT) converts the frequency-domain resource grid into a single time-domain digital signal.

![[PHY_Slide_15.png]]

2. **Digital-to-Analog Conversion (DAC):** The digital samples are converted into analog waveforms.
3. **RF Processing:** The waveforms are up-converted to the target radio frequency (RF) carrier band (such as Sub-6 GHz FR1 or mmWave FR2) and amplified for transmission across the physical antenna array elements.

---

## 📊 Summary of PHY Operations

The physical layer acts as the mathematical link engine converting logical blocks to analog waves:

![[PHY_Slide_16.png]]

* **Channel Inventory:** Defines downlink (PBCH, PDSCH, PDCCH) and uplink (PUSCH, PUCCH, PRACH) tracks.
* **LDPC Graph Selection:** Dynamically segments Transport Blocks into Code Blocks using Base Graph 1 or 2.
* **Rate Matching & Scrambling:** Adjusts coding density and randomizes bits to prevent inter-cell interference.
* **Precoding & Beamforming:** Maps spatial layers to virtual antenna ports to shape analog radio frequency beams.
