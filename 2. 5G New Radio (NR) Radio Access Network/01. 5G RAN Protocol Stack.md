# 01. 5G RAN Protocol Stack

To format, manage, and transmit user and signaling data over the air, the gNodeB (5G base station) and the User Equipment (UE) utilize a layered protocol stack. Similar to the internet's TCP/IP model, 5G NR divides its communication functions into specialized layers, split into two parallel stacks: the **User Plane** (for user data) and the **Control Plane** (for network signaling).

![[Materials/Slides/RAN_Slide_1.png]]

---

## 🏗️ Architectural Foundations & The Data Unit Flow

Every single layer in the 5G NR (New Radio) protocol stack provides a specialized service to the layer directly above it and consumes the services of the layer directly beneath it. As traffic travels down the transmitter stack, it undergoes continuous encapsulation:

* **SDU (Service Data Unit):** The unaltered data packet received from the layer *above*.
* **PDU (Protocol Data Unit):** The structural output package generated by the *current* layer after it processes the SDU and attaches its own specific control header ($H$).

Therefore, a **Layer N PDU** is handed down and immediately becomes the **Layer N-1 SDU** for the next layer down.

![[Materials/Slides/RAN_Slide_9.png]]

$$\text{Layer } N+1\text{ SDU} \longrightarrow \text{Layer } N \text{ processing} + \text{Header } (H) \longrightarrow \text{Layer } N\text{ PDU} \longrightarrow (\text{becomes the Layer } N-1\text{ SDU})$$

### Multi-Service Flow Example:
Incoming services (Google IP packets, Weather App IP packets, Voice VoIP packets) enter as **SDAP SDUs**. SDAP maps them and tacks on QoS headers to produce **SDAP PDUs**, which are passed down to PDCP as **PDCP SDUs** to be ciphered and compressed.

![[Materials/Slides/RAN_Slide_10.png]]

## 💎 Layer 2 (User Plane): The Data Wranglers

When you stream a video or make an internet request, the data travels through the **User Plane Protocol Stack**, which consists of four specialized sublayers within Layer 2 before hitting the physical hardware:

```
[ Application / IP Packet Data ]
               ↓
 1. SDAP (Service Data Adaptation Protocol) 
               ↓
 2. PDCP (Packet Data Convergence Protocol)
               ↓
 3. RLC  (Radio Link Control)
               ↓
 4. MAC  (Medium Access Control)
```

![[Materials/Slides/RAN_Slide_2.png]]

---

### 1. SDAP (Service Data Adaptation Protocol) — Layer 2
SDAP is a brand-new sublayer introduced exclusively in 5G NR to bridge the gap between how the Core Network and the Radio Access Network manage Quality of Service (QoS).

![[Materials/Slides/RAN_Slide_8.png]]

* **The Problem It Solves:** The 5G Core network groups user traffic into abstract **QoS Flows**, marking each packet with a **QFI (QoS Flow Identifier)**. However, the gNodeB base station can only transmit data across the air using physical pipelines called **DRBs (Data Radio Bearers)**.
* **Core Functions:**
  * **QoS Flow-to-DRB Mapping:** Maps multiple incoming QoS Flows from the Core network onto specific over-the-air Data Radio Bearers (DRBs).
  * **Flow Multiplexing:** SDAP can bundle different applications together. For instance, a Zoom call and a WhatsApp video call might come from the Core as two distinct QoS Flows, but because they share identical conversational-video demands (GBR QFI 2), SDAP multiplexes them into a single, high-priority DRB (e.g., DRB 2).
* **Uplink Mapping Methods:** When the phone transmits data back to the tower, its internal SDAP maps traffic using two distinct strategies:
  * **Explicit Mapping:** The gNodeB explicitly commands the phone via Radio Resource Control (RRC) signaling to bind a specific QFI to a specific DRB.
  * **Reflective Mapping:** A highly efficient 5G innovation. The phone's SDAP observes incoming downlink packets. If the network is delivering QFI $X$ over DRB $Y$, the phone automatically applies that exact same mapping for its uplink packets, bypassing unnecessary control plane signaling overhead.
* **The SDAP PDU Header:** SDAP prepends a small **1-byte (8-bit) header** to the packet:
  * **QFI (6 bits):** Carries the explicit QoS Flow Identifier (allowing up to 64 unique QoS flows per PDU session).
  * **RDI (1 bit):** Reflective QoS Flow-to-DRB Indicator (controls whether the phone should dynamically update its reflective routing database).
  * **RQI (1 bit):** Reflective QoS Indicator (signals whether to apply reflective tracking to the upper Core NAS layers).

> [!NOTE]
> For an in-depth exploration of QoS flows, 5QI matrices, and reflective mappings, see the dedicated study guide: [[02. 5G SDAP and QoS Architecture]].

### 2. PDCP (Packet Data Convergence Protocol) — Layer 2
The SDAP PDU is handed down to the PDCP layer, which treats it as a **PDCP SDU**. PDCP is responsible for system security, efficiency, and architectural routing.

![[Materials/Slides/RAN_Slide_7.png]]

![[Materials/Slides/PDCP_Slide_1.png]]

#### A. Core Functions of PDCP
PDCP acts as the master junction box for user plane and control plane data streams. It executes security, structural, and efficiency operations:

![[Materials/Slides/PDCP_Slide_2.png]]

#### B. Header Compression via RoHC (Robust Header Compression)
Modern internet data relies heavily on IPv4 (20-byte IP header + 20-byte TCP header = 40 bytes) or IPv6 (40-byte IP header + 8-byte UDP header + 12-byte RTP header = 60 bytes). For lightweight applications like Voice over 5G (VoNR) or transactional IoT alerts where the payload itself might only be 20 bytes, these massive headers can dwarf the actual data payload, crippling over-the-air spectrum efficiency.

![[Materials/Slides/PDCP_Slide_3.png]]

* PDCP compresses these headers down to a tiny, highly compressed **2-to-3 byte marker** using **RoHC** before wireless transmission.
* The receiving PDCP safely decompresses it back into a standard IP header before passing it to wired networks.

#### C. Ciphering and Integrity Protection (Security)
To prevent over-the-air eavesdropping and tampering, PDCP implements robust cryptographic algorithms at the bottom of the radio stack:

![[Materials/Slides/PDCP_Slide_4.png]]

* **Ciphering (Encryption):** Prevents over-the-air eavesdropping by encrypting user and control data streams. It is applied to both the User Plane (UP) and Control Plane (CP) using keys:
  * $K_{\text{RRC, enc}}$ for RRC signaling commands.
  * $K_{\text{UP, enc}}$ for User Plane application data.
  * $K_{\text{NAS, enc}}$ for Core Network signaling.
* **Integrity Protection:** Cryptographically guarantees data origin authentication, ensuring that data hasn't been intercepted, injected, or modified by an attacker. It is mandatory for the Control Plane (RRC/NAS) and optional for the User Plane, utilizing keys:
  * $K_{\text{RRC, int}}$ for RRC signaling commands.
  * $K_{\text{UP, int}}$ for User Plane application data.
  * $K_{\text{NAS, int}}$ for Core Network signaling.
* **Granular Security Inputs/Parameters:**
  * **Key:** 128-bit key ($K_{\text{RRC/UP/NAS, enc/int}}$).
  * **Count:** A 32-bit counter derived from the **PDCP Sequence Number (SN)** and the **Hyper Frame Number (HFN)**. It increments with each packet to prevent replay attacks.
  * **Bearer:** 5-bit Bearer identity (DRB or SRB index), ensuring different radio bearers use different keystreams even with the same keys.
  * **Direction:** 1-bit value (0 for Uplink, 1 for Downlink), preventing keystream reuse between TX and RX.
  * **Length / Message:** The plaintext block (for ciphering) or the actual message payload (for integrity).
* **Cryptographic Engine Operations:**
  * **Ciphering:** The inputs are fed into the **5G Encryption Algorithm (NEA)** to produce a *Keystream block*, which is then XORed with the plaintext block to output the *Ciphertext block*.
  * **Integrity:** The inputs are fed into the **5G Integrity Algorithm (NIA)** to produce a 32-bit **MAC-I (Message Authentication Code for Integrity)** or **NAS-MAC**, which is appended to the packet. The receiver calculates its own **XMAC-I / XNAS-MAC** and compares them; if they mismatch, the packet is silently discarded.

#### D. Routing and Duplication of Split Bearers (Multi-Connectivity)
In Non-Standalone (NSA) or Dual-Connectivity deployments, a single data stream needs to be split across two different physical base stations (the Master Cell Group / MCG under LTE and the Secondary Cell Group / SCG under 5G NR).

![[Materials/Slides/PDCP_Slide_5.png]]

* The PDCP layer acts as the master junction box. It handles **routing** (deciding which lower RLC/MAC layer to send packets through) and **packet duplication** (sending the identical packet down both cellular paths MCG and SCG simultaneously to guarantee ultra-reliable, low-latency **URLLC** delivery).
* The receiving PDCP identifies duplicate sequence numbers and discards the second copy, providing robust path diversity.

#### E. In-Sequence Delivery, Reordering, and Buffer Management
Because the wireless radio link can drop packets or route them across multi-connectivity paths, data packets frequently arrive out of order (e.g., Packet 1, 2, 4, then 3).

![[Materials/Slides/PDCP_Slide_6.png]]

* PDCP attaches an explicit **Sequence Number (SN)** to every PDU.
* If a packet arrives out of order, the receiving PDCP parks the incoming data inside a reordering buffer and starts a reordering timer (`t-Reordering`).
* It deliberately holds Packet 4 back until the delayed Packet 3 safely arrives. Once the sequence is complete, it flushes them sequentially to upper layers.
* If the timer expires before Packet 3 arrives, it assumes the packet was permanently lost, flushes Packet 4, and advances its window to prevent system stalling.

#### F. PDCP Processing Summary
The entire flow through the PDCP sublayer can be summarized by tracing the transmitter and receiver operations:

![[Materials/Slides/PDCP_Slide_7.png]]

* **Transmitter (SDU ➔ PDU):**
  $$\text{SDAP PDU (PDCP SDU)} \longrightarrow \text{Header Compression (RoHC)} \longrightarrow \text{Integrity Protection (MAC-I)} \longrightarrow \text{Ciphering (NEA)} \longrightarrow \text{Add PDCP SN Header} \longrightarrow \text{Routing / Duplication} \longrightarrow \text{PDCP PDU}$$
* **Receiver (PDU ➔ SDU):**
  $$\text{PDCP PDU} \longrightarrow \text{Reordering Buffer \& Duplicate Discarding} \longrightarrow \text{Deciphering (NEA)} \longrightarrow \text{Integrity Verification (Compare MAC-I \& XMAC-I)} \longrightarrow \text{Header Decompression (RoHC)} \longrightarrow \text{Remove PDCP Header} \longrightarrow \text{SDAP PDU (PDCP SDU)}$$

### 3. RLC (Radio Link Control) — Layer 2
The PDCP PDU becomes the **RLC SDU**. The RLC layer is the structural framing expert that bridges physical data constraints with logical pipelines.

![[Materials/Slides/RAN_Slide_6.png]]

* **Segmentation:** The lower MAC layer continuously calculates exactly how many bits can fit into the next instantaneous over-the-air transmission window based on current radio conditions. If an incoming upper-layer packet (RLC SDU) is too large for that allocation, the RLC layer slices and fragments the packet (**segmentation**) into perfectly sized pieces to fill the available radio frame capacity, adding an RLC header for tracking.
* **ARQ (Automatic Repeat Request) Error Correction:** RLC acts as an analytical safety net. Operating in Acknowledged Mode (AM), if a transmitted packet fragment gets corrupted over the air, the receiving RLC tracks the missing piece and issues an explicit retransmission request to the sender, ensuring zero data loss before it passes traffic up to PDCP.

---

### 4. MAC (Medium Access Control) — Layer 2
The RLC PDU becomes the **MAC SDU**. MAC is the ultimate traffic cop and resource distributor of the radio node.

![[Materials/Slides/RAN_Slide_5.png]]

* **Scheduling:** The physical airwaves are a shared medium. The gNodeB's centralized MAC scheduling algorithm determines exactly *which* devices get radio frequencies, *when* they get them, and *how much* power/bandwidth they are allowed to consume.
* **Multiplexing and Demultiplexing:** Takes different logical data channels (like control commands vs web browsing data), flattens them, and multiplexes them together into a unified structural payload (Transport Block).
* **HARQ (Hybrid Automatic Repeat Request):** Unlike RLC's software-driven ARQ, HARQ is a blindingly fast, hardware-level error correction loop operating at the very bottom of Layer 2. When a data block is sent, the receiving hardware checks it instantly using a Cyclic Redundancy Check (CRC). If a bit-level error is discovered, it fires an immediate NACK (Negative Acknowledgement) back to the transmitter's hardware, triggering an instant, microsecond-level physical retransmission.

## 📡 Layer 1: PHY (Physical Layer)

The MAC layer packages data into final units called **Transport Blocks** and hands them down to the **Physical Layer (PHY)**.

![[Materials/Slides/RAN_Slide_3.png]]

![[Materials/Slides/RAN_Slide_4.png]]

* **Role:** Pure execution over the airwaves.
* **Core Functions:** Takes digital bits and maps them into real physical analog radio frequency waveforms. It governs:
  * **Bit-level Channel Coding:** Utilizes **LDPC (Low-Density Parity-Check)** codes for data channels and **Polar codes** for control channels.
  * **Digital Modulation:** Maps bit sequences into complex analog constellation symbols (QPSK, 16QAM, 64QAM, 256QAM).
  * **MIMO & Spatial Processing:** Multi-antenna spatial multiplexing (Massive MIMO) to transmit multiple data streams concurrently.
  * **Dynamic Beamforming:** Focusing electromagnetic energy in precise directional beams toward specific UEs to maximize signal strength and range.

---

## 🚦 The Control Plane Protocol Stack (Network Signaling)

The network requires a dedicated, highly protected parallel stack to process commands, register devices, and maintain connections. The Control Plane utilizes the **exact same PHY, MAC, RLC, and PDCP layers** to guarantee over-the-air delivery, but bypasses the SDAP layer entirely.

![[Materials/Slides/RAN_Slide_11.png]]

Instead, it hosts two high-level control protocols at Layer 3:

### 1. RRC (Radio Resource Control) — Layer 3 (Radio Domain)
RRC operates strictly between the **User Equipment (UE)** and the **gNodeB base station**. It handles the configuration and state management of the radio links:

![[Materials/Slides/RAN_Slide_13.png]]

* **System Information Broadcasts:** The gNodeB constantly beams out critical master parameters (Master Information Block [MIB] and System Information Blocks [SIBs]) so any nearby smartphone can scan, discover the cell's identity (Cell ID), synchronize its internal timing clocks, and check local access tracking areas.
* **Connection Management:** Handles the formal setup, runtime maintenance, configuration modification, and teardown of signaling and data channels (Signaling Radio Bearers [SRBs] and Data Radio Bearers [DRBs]). Coordinates transitions between RRC States: **RRC_IDLE**, **RRC_INACTIVE**, and **RRC_CONNECTED**.
* **Measurements & Handovers:** Dictates when your phone should execute radio quality measurements of neighboring towers, evaluates the reports, and commands a seamless handover when you are moving out of range.
* **Paging:** When a data packet arrives for a device that has entered an energy-saving idle state, RRC blasts a waking broadcast signal across the local tracking area to alert the device to stand up and reconnect.

### 2. NAS (Non-Access Stratum) — Layer 3 (Core Domain)
NAS is the highest control layer. It handles end-to-end communication directly between the **UE** and the **AMF (Access and Mobility Management Function)** in the 5G Core Network.

*The gNodeB base station does not read, unpack, or process NAS messages; it acts merely as a transparent postman passing the messages through.*

![[Materials/Slides/RAN_Slide_12.png]]

* **Core Functions:** Governs user **Authentication** verifications, primary network **Registration** procedures upon device boot, security key management (deriving NAS keys), **Idle mode mobility procedures**, and session management initialization (such as requesting the Core network to provision a new PDU session and assign an IP address).

## 🏎️ Macro Example: Tracking a Packet’s Complete Downward Journey

To tie all of this information together into one comprehensive picture, let's trace a weather app data update as it is transmitted down to a phone:

1. **IP Layer:** A raw internet packet arrives at the gNodeB's user plane.
2. **SDAP Layer:** Reads the packet’s Core QoS profile, stamps it with a **QFI** within a **1-byte header**, maps it to the pre-configured Data Radio Bearer (DRB 2), and outputs an **SDAP PDU**.
3. **PDCP Layer:** Takes that packet as a PDCP SDU. It strips away the heavy 40-to-60 byte IP header using **RoHC**, applies **Integrity Protection**, runs **Ciphering** for encryption, assigns an explicit **Sequence Number (SN)**, and generates a **PDCP PDU**.
4. **RLC Layer:** Evaluates the buffer requirements, executing **Segmentation** to chop the data into fragments matching the over-the-air slot budget, and adds an RLC header for AM ARQ tracking.
5. **MAC Layer:** Gathers the RLC segments, **multiplexes** them into a single transport stream, attaches a scheduling wrapper, and maps them to **HARQ** hardware tracking blocks (generating a Transport Block).
6. **PHY Layer:** Receives the final **Transport Block**, processes the digital bits with LDPC channel coding, maps them into a complex QAM constellation, runs it through beamforming antenna matrices, and converts it into a **Physical Radio Wave** shot across the airwaves to the user.

The following matrix summarizes the complete 5G RAN protocol stack, showing how each layer treats User Plane data and Control Plane signaling:

| Layer | Protocol | Plane | Core Functions & Field Parameters | Data Unit |
| :--- | :--- | :--- | :--- | :--- |
| **Layer 3** | **NAS** | Control | Authentication, registration, session setup, mobility. Speaks directly to Core AMF. | Message |
| **Layer 3** | **RRC** | Control | Radio link setup, system broadcasts (MIB/SIBs), measurements, handovers, paging. | Message |
| **Layer 2** | **SDAP** | User | QoS Flow-to-DRB mapping, flow multiplexing. Header: 1 byte (6-bit QFI, 1-bit RDI, 1-bit RQI). | SDU/PDU |
| **Layer 2** | **PDCP** | Both | Header compression (RoHC), Ciphering (NEA, $K_{\text{enc}}$), Integrity protection (NIA, $K_{\text{int}}$, MAC-I), split bearer duplication/routing, reordering. | SDU/PDU |
| **Layer 2** | **RLC** | Both | Segmentation, reassembly, ARQ error correction. | SDU/PDU |
| **Layer 2** | **MAC** | Both | Dynamic scheduling, logical channel multiplexing, hardware-level HARQ (ACK/NACK). | SDU/Transport Block |
| **Layer 1** | **PHY** | Both | LDPC/Polar channel coding, digital modulation, massive MIMO, dynamic beamforming. | Transport Block/Radio Wave |
