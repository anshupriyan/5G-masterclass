# 01. 5G Core Network Identifiers

To route data, establish secure connections, manage mobility, and authenticate users, the 5G System relies on a highly structured hierarchy of network identifiers. These identifiers are divided into device-specific, subscription-specific, and temporary network runtime coordinates.

![[Materials/Slides/ID_Slide_1.png]]

---

## 🏷️ 1. Identifiers Classification

5G Core network identifiers are classified into three core pillars:

![[Materials/Slides/ID_Slide_2.png]]

1. **Device Identity (PEI):** Uniquely identifies the physical hardware itself (the terminal/phone).
2. **Subscription Identity (SUPI & SUCI):** Uniquely identifies the subscriber credentials stored on the SIM card.
3. **Temporary Network Identity (GUTI):** Globally unique temporary runtime identities assigned dynamically to protect user privacy.

![[Materials/Slides/ID_Slide_3.png]]

---

## 📱 2. Device Identity: Permanent Equipment Identifier (PEI)

Every physical device accessing the 5G network is assigned a **Permanent Equipment Identifier (PEI)**.

![[Materials/Slides/ID_Slide_4.png]]

### Key Characteristics:
* **Storage:** Stored permanently within the secure hardware environment of the User Equipment (UE) by the manufacturer.
* **Transmission:** Transmitted to the Core Network (specifically the AMF) securely during authentication and connection setup.
* **Emergency Usage:** If a device has no active SIM card, it can still register with the network for emergency services using its PEI.

### PEI Formats:
Depending on the device type, the PEI is formatted as one of the following:
1. **IMEI (International Mobile station Equipment Identity):** A 15-digit decimal number.
2. **IMEISV (International Mobile station Equipment Identity and Software Version Number):** A 16-digit decimal number containing software tracking details.

### The IMEI / IMEISV Structure & Digit Boundaries:

| Field Name | Abbreviation | Digit Size (IMEI) | Digit Size (IMEISV) | Technical Description |
| :--- | :--- | :--- | :--- | :--- |
| **Type Allocation Code** | **TAC** | 8 digits | 8 digits | Identifies the specific manufacturer, model, and assembly origin. |
| **Serial Number** | **SNR** | 6 digits | 6 digits | Unique serial number identifying the specific physical device. |
| **Check Digit** | **CD** | 1 digit | N/A (Omitted) | Calculated using the Luhn algorithm to check digit string integrity. |
| **Software Version Number** | **SVN** | N/A (Omitted) | 2 digits | Identifies the specific firmware/software version running on the device. |

## 💳 3. Subscription Identifiers: SUPI vs. SUCI

To authorize network access, the system must verify the subscriber's profile.

![[Materials/Slides/ID_Slide_5.png]]

5G NR introduces a critical security split for subscription identification over-the-air:

![[Materials/Slides/ID_Slide_6.png]]

### A. SUPI (Subscription Permanent Identity)
The **SUPI** is the permanent subscription identifier. It is the 5G equivalent of the 4G IMSI (International Mobile Subscriber Identifier). It is stored securely inside the Universal Subscriber Identity Module (USIM) on the SIM card.

#### SUPI Formats:
* **IMSI-based:** A 15-digit decimal string containing:
  * **Mobile Country Code (MCC):** 3 digits identifying the home country.
  * **Mobile Network Code (MNC):** 2 or 3 digits identifying the operator.
  * **Mobile Subscriber Identification Number (MSIN):** Up to 10 digits identifying the specific subscriber within that operator's network.
* **NAI (Network Access Identifier):** Formatted as a standard username/realm string (RFC 7542), commonly used in non-3GPP access (e.g. Wi-Fi).
  * *Example:* `0<IMSI>@wlan.mnc<MNC>.mcc<MCC>.3gppnetwork.org`

---

### B. SUCI (Subscription Concealed Identity) — The 5G Privacy Guard

In older networks like 4G LTE, the permanent IMSI was transmitted in plain text over-the-air during initial attach requests. This structural vulnerability allowed **"IMSI Catchers"** (stingrays) to intercept the plaintext IMSI, trace users, and compromise privacy.

![[Materials/Slides/ID_Slide_7.png]]

5G NR completely resolves this by introducing the **Subscription Concealed Identity (SUCI)**:
1. **Public Key Cryptography:** The USIM card stores the **Home Network Public Key** provisioned by the operator.
2. **ECIES Encryption Scheme:** The UE uses the **Elliptic Curve Integrated Encryption Scheme (ECIES)** to encrypt the sensitive **MSIN** portion of the SUPI. The Mobile Country Code (MCC) and Mobile Network Code (MNC) **remain unencrypted (plaintext)**.
3. **Visited Network Routing:** Because the MCC and MNC are plaintext, a visited network (VPLMN) can read them and route the registration request to the correct Home PLMN (HPLMN).
4. **Decryption at UDM:** The SUCI is transmitted across the radio links. The base station (gNB) and AMF cannot decrypt it. The SUCI is forwarded to the **Unified Data Management (UDM)** in the home core network. The UDM decrypts the encrypted MSIN portion using the corresponding home network private key, recovering the plaintext SUPI.
5. **Privacy Assured:** The plaintext SUPI is *never* transmitted over the air, completely eliminating IMSI-catching vulnerabilities.

## 🔄 4. Globally Unique Temporary Identifier (5G-GUTI)

To prevent tracking of active devices, the network assigns a temporary, dynamically changing identifier called the **5G-GUTI (Globally Unique Temporary Identifier)** once initial registration is complete.

![[Materials/Slides/ID_Slide_8.png]]

The 5G-GUTI is constructed mathematically from the Global AMF Identifier and a temporary subscriber number:
$$\text{5G-GUTI} = \text{GUAMI} + \text{5G-TMSI}$$

### 1. GUAMI (Globally Unique AMF Identifier)
The GUAMI uniquely identifies a specific Access and Mobility Management Function (AMF) instance in the 5G Core. It consists of:
* **MCC (Mobile Country Code):** Home country identifier (3 digits).
* **MNC (Mobile Network Code):** Home operator identifier (2 or 3 digits).
* **AMF Identifier (AMFI):** A 24-bit field broken down as:
  * **AMF Region ID (8 bits):** Identifies a specific geographic region.
  * **AMF Set ID (10 bits):** Identifies a cluster/set of AMFs servicing that region.
  * **AMF Pointer (6 bits):** Points directly to the specific AMF node holding the UE context.

### 2. 5G-TMSI (Temporary Mobile Subscriber Identity)
A **32-bit temporary number** assigned dynamically by the AMF. The UE uses the 5G-GUTI (and specifically the 5G-TMSI) in all subsequent uplink signaling messages to hide its identity.
