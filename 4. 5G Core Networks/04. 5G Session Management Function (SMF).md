# 04. 5G Session Management Function (SMF)

While the AMF manages access, security, and mobility, the **Session Management Function (SMF)** is the control plane engine responsible for establishing, maintaining, modifying, and releasing the user-plane data paths.

![[Materials/Slides/SMF_Slide_1.png]]

---

## 🗺️ 1. Core Session Management Principles

In 5G, the control plane is completely isolated from the user plane.

![[Materials/Slides/SMF_Slide_2.png]]

![[Materials/Slides/SMF_Slide_3.png]]

* **Session Management (SM):** The control plane procedures required to set up data networks.
* **The Indirect Path:** The SMF cannot communicate directly with the UE over-the-air. Instead, all session management NAS messages are packaged in an envelope, sent to the AMF over the N11 interface, and the AMF relays them to the UE via the N1 interface.

---

## 📲 2. Defining the PDU Session

To transmit data, the UE must establish a **PDU (Packet Data Unit) Session**.

![[Materials/Slides/SMF_Slide_4.png]]

### PDU Session Concept:
A PDU Session is a logical, secure control and user-plane connection between the UE and the exit port of the **User Plane Function (UPF)** toward a specific external **Data Network (DN)** (such as the Internet, IMS, or a corporate intranet). A single UE can maintain multiple PDU Sessions simultaneously to different Data Networks.

### Core Properties of a PDU Session:

![[Materials/Slides/SMF_Slide_5.png]]

* **PDU Session Identifier:** A unique numeric identifier generated by the UE.
* **Slice Identifier (S-NSSAI):** Single Network Slice Selection Assistance Information. Consists of:
  * **SST (Slice/Service Type):** An 8-bit field specifying the baseline slice behavior (e.g., eMBB, URLLC, mMTC).
  * **SD (Slice Differentiator):** A 24-bit optional field enabling operators to differentiate between services of the same SST.
* **Data Network Name (DNN):** The 5G equivalent of the APN (Access Point Name), specifying the target external network.
* **PDU Session Type:** The protocol format used to carry packets (IP, Ethernet, or Unstructured).
* **SSC Mode:** The Service and Session Continuity mode assigned to ensure session persistence during mobility.

## 🩻 3. Three 5G PDU Session Types

Unlike 4G LTE which only supported IP-based sessions, 5G NR introduces three distinct PDU Session Types to support diverse vertical industries:

![[Materials/Slides/SMF_Slide_6.png]]

![[Materials/Slides/SMF_Slide_7.png]]

### A. IP-based PDU Session
* **Formats:** Supports **IPv4**, **IPv6**, or dual-stack **IPv4v6**.
* **IP Allocation Methods:** The SMF is strictly responsible for dynamic IP address allocation. It provisions the UE during session setup using:
  * **DHCPv4 / DHCPv6:** Dynamic Host Configuration Protocol.
  * **IPv6 SLAAC:** Stateless Address Autoconfiguration, allowing the UE to construct its own IP based on router advertisements.

### B. Ethernet PDU Session (5G Specific)
* **Concept:** Connects the UE directly to a Layer 2 Ethernet Data Network, carrying raw Ethernet frames (MAC frames) over-the-air.

![[Materials/Slides/SMF_Slide_8.png]]

* **Routing & Anchoring Switch Logic:**
  * The SMF and UPF act as a logical Layer 2 switch, performing **MAC address learning** to route packets based on source and destination MAC addresses in local forwarding tables.
  * The external Data Network can dynamically provide the MAC address during session setup.
  * Packets are encapsulated and routed via a point-to-point tunnel (e.g. GRE or UDP/IPv6) established between the UPF and the external Data Network.
  * Supports SSC Modes 1 and 2.

### C. Unstructured PDU Session (IoT Specific)
* **Concept:** User data is treated as raw, unstructured bits with zero IP or Ethernet header overhead.

![[Materials/Slides/SMF_Slide_9.png]]

* **Applications:** Used to carry lightweight IoT protocols (such as MQTT-SN, CoAP, or 6LoWPAN) directly without header bloat, maximizing spectral efficiency.
* **Tunneling:** Packets are tunneled cleanly over N6 to the DN using point-to-point UDP or GRE tunneling.
* **QoS:** No quality of service differentiation is performed; data is routed as best-effort unstructured streams. Supports SSC Modes 1 and 2.

## 🏎️ 4. Session Continuity and PDU Session Anchors

When a UE moves across cells, its data paths must be redirected. The anchor point where the User Plane Function exits to the external Data Network is called the **PDU Session Anchor (PSA)**.

![[Materials/Slides/SMF_Slide_10.png]]

![[Materials/Slides/SMF_Slide_11.png]]

To manage this redirection, 5G NR defines three **Service and Session Continuity (SSC) Modes**:

![[Materials/Slides/SMF_Slide_12.png]]

### 1. SSC Mode 1 (Continuous Anchoring)
* **Mechanism:** The PDU Session Anchor UPF assigned during session establishment remains fixed, regardless of how far the UE moves.
* **IP Address:** Mapped strictly to a **single IP address (IP1)** that remains completely static.
* **Usage:** Best for steady, long-running sessions (such as VoIP or file downloads) where IP stability is mandatory.

### 2. SSC Mode 2 ("Break-before-make")
* **Mechanism:** As the UE moves away from its original anchor, the network decides to optimize path routing by relocating the anchor UPF closer to the UE's new location.
* **IP Address:** The network **releases/tears down the existing PDU session and IP1** before establishing a new PDU session anchor and assigning a **new IP address (IP2)**.
* **Usage:** Used when latency optimization is prioritized and a temporary connection break during handover is acceptable.

### 3. SSC Mode 3 ("Make-before-break")
* **Mechanism:** High-reliability, seamless transition. The network establishes a new PDU session anchor UPF and assigns a **second IP address (IP2)** *before* releasing the original session and **IP1**.
* **Dual-IP Connectivity:** The UE temporarily holds **two active IP addresses (IP1 and IP2) simultaneously**. Data is seamlessly shifted to the new path, and only then is the old IP1 session released.
* **Usage:** Used to guarantee zero data loss or session interruption during anchor relocation.

## 📊 Summary of SMF Operations

The SMF acts as the dynamic control center for all user data pathways:

![[Materials/Slides/SMF_Slide_13.png]]

![[Materials/Slides/SMF_Slide_14.png]]

* **Session Lifecycle:** Governs the setup, modification, and teardown of IP, Ethernet, and Unstructured PDU Sessions.
* **IP Address Provisioning:** Dynamically allocates and manages IPv4, IPv6, and dual-stack IPv4v6 pools.
* **UPF & PCF Selection:** Analyzes slice configurations (S-NSSAI) and network conditions to select the optimal UPF anchor and PCF instance.
* **Billing and Charging:** Configures and controls online and offline billing rules inside the UPF using the PFCP protocol.
