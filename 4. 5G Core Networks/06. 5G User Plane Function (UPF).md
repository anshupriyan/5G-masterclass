# 06. 5G User Plane Function (UPF)

In the 5G Core (5GC) Control and User Plane Separation (CUPS) architecture, all user data traffic is completely isolated from control signaling. The **User Plane Function (UPF)** is the workhorse of the user plane, responsible for all packet routing, forwarding, encapsulation, and QoS enforcement.

![[UPF_Slide_1.png]]

---

## 🏗️ 1. Main UPF Mandates and Interface Anchoring

The UPF resides at the boundary between the RAN and external networks, executing several high-speed data plane operations:

![[UPF_Slide_2.png]]

* **PDU Session Anchor (PSA):** Acts as the final user-plane exit point toward external networks, hiding UE mobility from the internet. When a UE moves across cells, the UPF anchor remains stable to keep user IP sessions active.
* **External Network Interconnection:** Connects directly with external Data Networks (DNs) over the **N6 interface**.
* **Mobility Anchoring:** Anchors data paths during handovers. If a UE transitions to a new cell under a new gNodeB, the UPF seamlessly redirects data paths over the **N9 interface** (UPF-to-UPF tunnel) or **N3 interface** (gNB-to-UPF).
* **Packet Buffering:** If downlink data arrives for a device that is currently sleeping in `CM-IDLE` mode, the UPF caches/buffers the packets and triggers a buffer notification to the SMF/AMF over N4, which prompts the AMF to initiate paging over N2.
* **QoS Marking:** Maps and applies explicit QoS Flow Identifiers (QFIs) on all user packets before forwarding them to the RAN.
* **Usage Reporting:** Counts bytes and packets to generate charging records and reports usage data to the SMF.

## ⚡ 2. The PFCP Protocol and SMF Rule Controls (Detailed Fields)

The UPF does not make routing decisions independently. It is a stateless, pure execution engine controlled strictly by the Session Management Function (SMF) over the **N4 interface**.

![[UPF_Slide_3.png]]

This control relationship is governed by the **PFCP (Packet Forwarding Control Protocol)** (3GPP TS 29.244). The SMF installs explicit lookup and action rules in the UPF using five distinct PFCP control rule sets:

---

### A. PDR (Packet Detection Rules)
* **Role:** Tells the UPF how to detect and classify incoming packets.
* **Granular Fields inside PDR:**
  * **Packet Detection Information (PDI):** The container holding all matching parameters:
    * **Source Interface:** Identifies the origin of the packet (Access, Core, SGi-LAN/N6, CP-function).
    * **Local F-TEID (Fully Qualified Tunnel Endpoint Identifier):** The GTP-U tunnel IP and TEID allocated by the UPF for this specific flow.
    * **UE IP Address:** The IP assigned to the device.
    * **SDF Filter:** The standard IP 5-tuple (Source IP, Destination IP, Source Port, Destination Port, Protocol) used to identify application flows.

---

### B. FAR (Forwarding Action Rules)
* **Role:** Tells the UPF what to do with a packet once it matches a PDR.
* **Granular Fields inside FAR:**
  * **Action:** Commands the UPF to `FORWARD` (route to destination), `DROP` (discard), `DUPLICATE` (for law enforcement interception or URLLC redundancy), or `BUFFER` (cache for sleeping devices).
  * **Destination Interface:** The target exit interface (Access, Core, SGi-LAN/N6).
  * **Outer Header Creation:** The GTP-U encapsulation headers (Destination IP, TEID) that the UPF must append to the packet before sending it to the target gNodeB or UPF.

---

### C. BAR (Buffering Action Rules)
* **Role:** Configures buffering behaviors for inactive UEs (in `CM-IDLE` state).
* **Granular Fields inside BAR:**
  * **Buffering Action ID:** Unique identifier for the buffer pool.
  * **Downlink Data Notification Delay:** How long the UPF should wait before alerting the SMF that data has arrived (to prevent paging storm issues).
  * **Suggested Buffering Packets Count:** The maximum number of packets the UPF should cache for this session before discarding overflows.

---

### D. QER (QoS Enforcement Rules)
* **Role:** Enforces quality of service limits on the data flow.
* **Granular Fields inside QER:**
  * **Gate Status:** Open (allow) or Closed (block).
  * **MBR (Maximum Bit Rate):** The maximum permitted uplink/downlink throughput limits.
  * **GBR (Guaranteed Bit Rate):** The guaranteed bandwidth reserved for this flow.
  * **QFI (QoS Flow Identifier):** The explicit 5G QoS tag that the UPF must write directly into the GTP-U outer header.
  * **Reflective QoS:** Activation flag instructing the UE to replicate the downlink QoS rules for its uplink packets.

---

### E. URR (Usage Reporting Rules)
* **Role:** Measures and reports data volume and duration.
* **Granular Fields inside URR:**
  * **Measurement Method:** Dictates what to measure (Volume in bytes, Time in seconds, or Event counts).
  * **Reporting Triggers:** Thresholds that trigger a report to the SMF (Volume Threshold, Time Threshold, Periodic Report, or Start/Stop of traffic).
  * **Volume/Time Quota:** Prepaid limits allowed for the session.
